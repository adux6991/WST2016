I choose Python Django Framework.

First of all, I have to install Django without sudo. I find pip install --user extremely useful and succeed after a lot of effort.

Second, in order to deploy Django in Apache, I manually build mod_wsgi.so and load this module in httpd.conf. Thus when Apache detects /django in URL, Django will be called to handle it. Since the built-in web server in Django is too weak to be deployed in production environment, I am not able to use Reverse Proxy or Load Balance.

At last, the most important part is coding. Django is a MVC framework, but the names are different:

  • Model:       models.py
  • View:          templates/xxx.html
  • Controller: urls.py & views.py

Here is what every file does:

  • models.py: define a class Guest with four attributes (name, age, gender, email) and interact with database automatically.
  • urls.py: associate URL with functions in views.py. For example, when the URL /django/polls/query/ comes, the function query in views.py is called.
  • views.py: control the database and send results to templates. For example, the function query fetches all Guest entries from the database and send them to templates/query.html.
  • templates: define how data is presented, act as View in MVC.

Homework 4

Write a brief essay outlining the differences between AWT and Swing graphics.

  • AWT(Abstract Window Toolkit) is a Java interface to native system GUI code present in your OS. It will not work the same on every system, although it tries.
  • Swing is a more-or-less pure-Java GUI. It uses AWT to create an operating system window and then paints pictures of buttons, labels, text, checkboxes, etc., into that window and responds to all of your mouse-clicks, key entries, etc., deciding for itself what to do instead of letting the operating system handle it. Thus Swing is 100% portable and is the same across platforms (although it is skinnable and has a "pluggable look and feel" that can make it look more or less like how the native windows and widgets would look).
